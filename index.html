Project Overview

プロジェクト概要

本プロジェクトは、IT企業 NEXUS Inc. のコーポレートサイトを 新規制作するものです。先進的なIT企業にふさわしいブランド表現と、 採用・問い合わせといったビジネス成果に直結するサイト構造の両立を目指します。

本提案の位置づけ

「ヒアリング → 戦略設計 → 提案 → 制作 → 公開」という実務フローのうち、 本書は戦略設計から提案までをまとめたものです。サイトマップ・ デザインコンセプト・スケジュール・概算見積もりまでを一冊に集約し、 意思決定に必要な情報を一度に確認いただける構成としています。

Current Issues

現状課題

現行サイトを調査した結果、ブランド表現・コンテンツ・技術面の3領域で 課題が確認されました。これらはいずれも、採用・問い合わせの成果に 直接影響する重要な論点です。

確認された3つの課題

ISSUE 01

ブランド表現が事業の先進性と乖離している

デザインに古さがあり、AI・最新技術を扱う企業という第一印象を与えられていない。 競合他社と比較したとき、技術力が伝わりにくい。

TOP直帰率
72%
平均滞在時間
43秒
ISSUE 02

採用・問い合わせへの導線が弱い

サービス内容や実績は掲載されているが、「相談する」「応募する」という 次のアクションへの誘導が不足し、訪問者が離脱しやすい。

問い合わせCVR
0.4%
Service→Contact 離脱率
78%
ISSUE 03

スマホ対応・表示速度に改善の余地がある

モバイルでの閲覧体験が最適化されておらず、表示速度も遅い。 SEO・ユーザー体験の両面で機会損失が発生している。

モバイル LCP
4.8秒
モバイル比率
63%

※ 上記の数値は、GA4・PageSpeed Insightsを用いた現状調査に基づく想定値です（業界平均：CVR 1.2% / モバイル LCP 2.5秒以下が目安）。

課題に対する解決の方向性

ブランド表現黒×青紫グラデーションの先進的なビジュアルで刷新滞在時間 2分以上
導線設計各ページにCTAを配置し、Contactへ自然に誘導CVR 1.0%以上
技術・速度軽量な静的サイト構成＋レスポンシブ設計で最適化モバイル LCP 1.8秒以下

Tech Stack

使用技術

フレームワークやビルドツールを使わず、純粋なHTML / CSS / JavaScript のみで構築します。軽量・高速・保守しやすいことを重視した技術選定です。

技術スタック

カテゴリ採用技術
マークアップHTML5（セマンティックなマークアップ）
スタイリングCSS3（CSS変数 / Grid / Flexbox / clamp()）
スクリプトVanilla JavaScript（ES6+、ライブラリ不使用）
フォントGoogle Fonts（Space Grotesk / Noto Sans JP）
公開環境GitHub Pages（静的ホスティング）
バージョン管理Git / GitHub

ファイル構成

project/ ├── index.html ├── assets/images/ 画像（preview-mockup.png ほか） ├── css/ reset.css / style.css / pages.css ├── js/ main.js └── pages/ service / works / company ほか 6ページ

技術選定の理由

  • 軽量・高速：余計な依存がなく、表示速度を最適化できる
  • 保守性：CSS変数で色・余白を一元管理、変更が容易
  • 公開の手軽さ：GitHub Pagesで無料・短時間で公開可能

なぜフレームワークを採用しないか

React / Next.js のようなフレームワークは、本案件の規模・更新頻度に対して過剰設計と判断しました。 下記の前提に照らし、Vanilla JavaScript による静的構成を最適解として採用しています。

判断軸本案件の前提結論
サイト規模全7ページの静的サイトSPA構成は不要
更新頻度ブログ：月1〜2本／その他：四半期更新ビルド工程の負荷が見合わない
運用担当非エンジニアの広報担当を想定学習コストの低い構成を優先
パフォーマンス初期表示速度を最重要視（LCP 1.8秒以下）JSバンドルを最小化したい

※ 将来的に更新頻度が大幅に増える場合は、microCMS等のヘッドレスCMSを連携した Jamstack 構成への段階的移行が可能です。

本提案で目指したこと

本提案では、ブランド表現の刷新だけでなく、
「問い合わせ・採用につながる導線設計」、
「クライアント自身が継続運用できる体制」、
「公開後にKPIで効果を測り、改善を回す仕組み」まで含めて、
"作って終わり"にならないコーポレートサイトの設計を目指しました。

企画から公開、そしてその先の運用まで——
一貫した責任を持って、伴走させていただきます。

本提案のまとめ

  • 先進的IT企業にふさわしいブランド表現で、現状の課題を解決
  • ターゲットの行動に沿った全7ページ構成で、成果につなげる
  • 軽量・高速な静的サイトとして、約8週間・概算100万円で制作
  • 公開後はKPIに基づき、月次レポートで継続的に改善
  • 企画 → 設計 → デザイン → 実装 → 公開 → 運用までを一貫して担当
