Project Overview

プロジェクト概要

本プロジェクトは、IT企業 NEXUS Inc. のコーポレートサイトを 新規制作するものです。先進的なIT企業にふさわしいブランド表現と、 採用・問い合わせといったビジネス成果に直結するサイト構造の両立を目指します。

本提案の位置づけ

「ヒアリング → 戦略設計 → 提案 → 制作 → 公開」という実務フローのうち、 本書は戦略設計から提案までをまとめたものです。サイトマップ・ デザインコンセプト・スケジュール・概算見積もりまでを一冊に集約し、 意思決定に必要な情報を一度に確認いただける構成としています。

Current Issues

現状課題

現行サイトを調査した結果、ブランド表現・コンテンツ・技術面の3領域で 課題が確認されました。これらはいずれも、採用・問い合わせの成果に 直接影響する重要な論点です。

確認された3つの課題

ISSUE 01

ブランド表現が事業の先進性と乖離している

デザインに古さがあり、AI・最新技術を扱う企業という第一印象を与えられていない。 競合他社と比較したとき、技術力が伝わりにくい。

ISSUE 02

採用・問い合わせへの導線が弱い

サービス内容や実績は掲載されているが、「相談する」「応募する」という 次のアクションへの誘導が不足し、訪問者が離脱しやすい。

ISSUE 03

スマホ対応・表示速度に改善の余地がある

モバイルでの閲覧体験が最適化されておらず、表示速度も遅い。 SEO・ユーザー体験の両面で機会損失が発生している。

課題に対する解決の方向性

課題本提案での解決アプローチ
ブランド表現黒×青紫グラデーションの先進的なビジュアルで刷新
導線設計各ページにCTAを配置し、Contactへ自然に誘導
技術・速度軽量な静的サイト構成＋レスポンシブ設計で最適化

Tech Stack

使用技術

フレームワークやビルドツールを使わず、純粋なHTML / CSS / JavaScript のみで構築します。軽量・高速・保守しやすいことを重視した技術選定です。

技術スタック

カテゴリ採用技術
マークアップHTML5（セマンティックなマークアップ）
スタイリングCSS3（CSS変数 / Grid / Flexbox / clamp()）
スクリプトVanilla JavaScript（ES6+、ライブラリ不使用）
フォントGoogle Fonts（Space Grotesk / Noto Sans JP）
公開環境GitHub Pages（静的ホスティング）
バージョン管理Git / GitHub

ファイル構成

project/ ├── index.html ├── assets/images/ 画像（preview-mockup.png ほか） ├── css/ reset.css / style.css / pages.css ├── js/ main.js └── pages/ service / works / company ほか 6ページ

技術選定の理由

  • 軽量・高速：余計な依存がなく、表示速度を最適化できる
  • 保守性：CSS変数で色・余白を一元管理、変更が容易
  • 公開の手軽さ：GitHub Pagesで無料・短時間で公開可能

Schedule

制作スケジュール

キックオフから公開まで、全5フェーズ・約8週間を想定しています。 各フェーズ末に成果物を共有し、認識のずれを早期に解消しながら進行します。

全体工程

フェーズ期間主な作業内容
1. 設計第1〜2週要件整理・サイトマップ確定・ワイヤーフレーム作成
2. デザイン第3〜4週デザインコンセプト確定・主要ページのデザイン制作
3. 実装第5〜6週HTML / CSS / JSによるコーディング・レスポンシブ対応
4. 検証第7週表示確認・動作テスト・修正対応
5. 公開第8週GitHub Pagesへのデプロイ・最終確認・納品
